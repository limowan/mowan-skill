# 业务逻辑安全清单

业务逻辑漏洞无法被 WAF 或通用安全工具检测，因为它们不违反任何技术规则——只是违反了业务规则。

---

## 支付与资金安全

### 检查项

1. **金额篡改**
   - 订单金额是否在前端传递（可被篡改为 0.01 或负数）
   - 服务端是否根据商品 ID 重新计算金额（而非信任前端传来的价格）
   - 折扣/优惠券计算是否在服务端完成

2. **支付回调验证**
   - 支付成功回调是否验签（支付宝 RSA 验签、微信签名校验）
   - 是否校验了回调中的订单号和金额与本地记录一致
   - 是否防止了回调重放（幂等处理）

3. **订单状态机**
   - 订单状态转换是否有严格的状态机约束
   - 是否可以从"已取消"状态直接跳到"已支付"
   - 是否可以对同一订单重复触发支付成功逻辑

4. **退款安全**
   - 退款是否需要额外的权限验证（2FA、审批流）
   - 退款金额是否可以超过原订单金额
   - 退款后权益是否正确回收

5. **货币精度**
   - 金额计算是否使用了浮点数（可能导致精度丢失）
   - 是否统一使用分为单位的整数计算

## 竞态条件 (Race Condition)

### 检查项

1. **余额/库存扣减**
   - 扣减操作是否使用了数据库事务
   - 是否使用了乐观锁（版本号）或悲观锁（SELECT FOR UPDATE）
   - 并发请求是否可能导致超卖/余额变负

2. **优惠券/兑换码**
   - 一次性优惠券是否可以被并发请求多次使用
   - 核销操作是否有原子性保证

3. **检测方法**
   - Grep 关键业务操作（扣款、发放、核销）
   - 检查是否在事务内完成"检查 + 操作"
   - 如果是"先查询再更新"的模式（TOCTOU），标记为潜在竞态

### 判定标准

- 支付/余额相关的竞态条件 → Critical
- 优惠券/积分的竞态条件 → High
- 非资金相关的竞态条件 → Medium

## 权限与越权

### 检查项

1. **水平越权（同级用户间）**
   - 用户 A 能否通过修改 ID 参数访问用户 B 的数据
   - 数据查询是否始终带有当前用户的过滤条件
   - 批量操作接口是否校验了每条数据的归属

2. **垂直越权（跨角色）**
   - 普通用户能否直接调用管理员接口
   - 权限校验是否在每个接口都执行（而非仅在前端隐藏按钮）
   - 是否存在"隐藏"的管理接口（未在前端展示但可直接访问）

3. **功能级越权**
   - 免费用户能否访问付费功能的 API
   - 试用期结束后 API 是否仍可访问
   - 权限变更后是否立即生效（缓存问题）

## 流程绕过

### 检查项

1. **步骤跳过**
   - 多步骤流程（注册→验证→激活）是否可以直接跳到最后一步
   - 是否可以通过直接调用后续步骤的 API 绕过前置检查
   - 验证码验证是否可以被跳过

2. **参数篡改**
   - 隐藏字段（如 `role=admin`、`is_vip=true`）是否可以被客户端修改
   - 批量赋值（Mass Assignment）：是否接受了不应由用户设置的字段
   - 数量/次数参数是否可以设为负数或极大值

3. **时序绕过**
   - 限时活动是否只在前端做时间校验
   - 服务端是否独立校验时间条件
   - 是否可以通过修改请求时间戳绕过

## 速率限制与滥用

### 检查项

1. **API 滥用**
   - 高成本操作（AI 调用、邮件发送、短信发送）是否有速率限制
   - 限制是基于什么维度（IP、用户、API Key）
   - 是否可以通过分布式请求绕过 IP 限制

2. **资源耗尽**
   - 是否可以创建大量资源（账号、订单、文件）耗尽存储
   - 复杂查询是否可以被滥用导致数据库负载过高
   - 文件上传是否可以耗尽磁盘空间

3. **爬虫/数据抓取**
   - 公开数据接口是否有分页限制
   - 是否可以通过遍历 ID 批量获取数据
   - 是否有反爬机制（但注意不要误伤正常用户）

---

## 数据导出与批量操作

### 检查项

1. **导出权限**
   - 数据导出是否有权限控制
   - 导出范围是否限制为当前用户有权访问的数据
   - 是否有导出量限制

2. **批量删除/修改**
   - 批量操作是否校验了每条数据的权限
   - 是否有操作数量上限
   - 是否有确认机制或冷却期

---

## 邀请与分享

### 检查项

1. **邀请链接**
   - 邀请 token 是否可预测
   - 邀请是否有使用次数限制
   - 邀请是否有过期时间
   - 是否可以通过邀请链接获得超出预期的权限

2. **分享链接**
   - 分享链接是否泄露了不应公开的数据
   - 分享权限是否可以被提升（只读 → 编辑）
   - 分享是否可以被撤销
