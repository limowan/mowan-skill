# 卡片导航提示词

本文件包含 3 种卡片导航样式的生成指令。当用户开启卡片导航时，从这里取对应样式的提示词拼入 AI 请求。

---

## 通用基础规则（所有导航样式共享）

```
允许为了生成目录而自主归纳摘要文字（享有零改字豁免权）。
⚠️ 导航区域的位置绝对固定：必须作为整篇文章 HTML 输出的第一个元素，放在所有正文内容、标题、引言之前。绝对不能把导航区域插入到文章中间或末尾。
整个导航区域的配色必须跟随当前风格的色板体系。
```

---

## 1. 滑动卡片（slide-cards）—— 默认样式

```
📌 原则上强制生成【滑动卡片式】目录导航区：
{通用基础规则}
头部标题行固定在目录区域顶部，不随卡片一起横向滑动。头部标题行使用左右两端布局：左侧放 emoji + N PARTS + CONCLUSION（N=实际章节数），右侧放"👉 滑动查看"引导文字。头部标题行使用 table 单行两列实现左右分布。
头部标题行必须放在 overflow-x:auto 滑动容器的外面，确保标题行始终可见。
每个目录项使用 Part 01、Part 02... 的编号格式，最后一个目录项固定使用 Last 作为编号。
不要生成额外的引导卡片——第一个卡片就是 Part 01。
第一个目录项使用比主题色更深的颜色作为背景以形成视觉锚点；其余目录项使用浅色背景。
每个目录项的副标题由 AI 自行从章节内容中总结 1-3 个核心关键词或简短概括。
整个导航区域使用双层 section 容器模拟边框效果（外层浅色背景 + 内层白色背景 + 间隙形成边框感）。不要使用单层 border（公众号可能剥离最外层 border）。
卡片区域使用 overflow-x:auto 和 white-space:nowrap 实现横向滑动，内部卡片使用 display:inline-block。
⚠️ 导航区域禁止使用灰色背景，外层容器背景必须是主题色极浅版本。
```

## 2. 仪表盘（dashboard）

```
📌 原则上强制生成【仪表盘式】目录导航区：
{通用基础规则}
仪表盘样式要求：整体是一个圆角矩形卡片（浅色背景），内部采用扁平化设计。
顶部栏左侧有小圆点+英文模块名，右侧有水平细线延伸；右下方放置预估阅读时长信息（由 AI 根据文章长度自行评估）。
中间核心区域：左侧展示起点状态（大号加粗中文）；中间上方是圆角胶囊样式的章节数量标签，下方是关键词文本；右侧展示终点状态。
下方是一排横向排列的圆角矩形按钮，代表各章节核心内容概括。按钮必须均匀分布占满整行宽度。第一个按钮用强调色背景+白色文字，其余用常规主题色。
底部栏显示品牌标注和从文章中提取的精华金句。
⚠️ 公众号兼容：所有 table 设置 border-collapse:collapse，td 设置 border:none。所有颜色跟随当前风格色板。
```

---

## 3. 标题卡片（title-card）

```
📌 原则上强制生成【标题卡片式】目录导航区：
{通用基础规则}
标题卡片样式要求：整体是一个圆角矩形卡片。
顶部栏：左侧是主题色背景的圆角小标签（白色英文文本），下方单独放当天日期（YYYY/MM/DD 格式）。右侧可放浅灰色英文副标题或栏目名。
中间大标题区：两行大号加粗文字。第一行用主题色显示主关键词；第二行用辅助色显示副关键词。
底部金句栏：较小的深灰色中文副标题或金句，核心关键词用主题色背景高亮。
所有颜色跟随当前风格色板，使用内联 style 实现。
```
