# 引导关注卡片提示词

当用户开启「引导关注卡片」时，将本文件的提示词拼入 AI 请求的末尾。

---

## 完整提示词

```
📌 原则上强制生成底部引导关注卡片：
位置绝对固定：必须作为整篇文章 HTML 输出的最后一个元素，放在所有正文、图片、结尾内容之后。绝对不能插入到文章中间。
外层关注卡片容器必须显式写成 <section data-follow-card="true" ...>；按钮组 table 必须写成 <table data-follow-card-actions="true" ...>；三个按钮 td 分别写 data-follow-card-action="like|read|share"。
动态主题色必须严格继承当前这篇排版已经采用的主主题色，不要额外发明新颜色。
设计风格：现代极简，大面积留白，依靠轻微软阴影构建层级，视觉焦点引导到"转发"按钮。
整体外层容器：纯白圆角矩形卡片，1px 极浅灰边框，宽裕内边距。
顶部引导文案固定使用：{followCardLeadText}（默认："既然看到这里了，随手点赞、在看、转发三连吧"）
核心按钮组必须是一行三列横向布局，三个按钮等宽等距水平居中分布，文案严格使用"点赞""在看""转发"。绝对不允许生成纵向堆叠版本。
按钮组必须用 table 实现单行三列：一个 table，只有一行 tr，三个 td，每个 td 宽度约 33.33%。
按钮组 table 自身 border:none;background:transparent，不要给整个区域额外套描边外框。
图标规则：不使用 SVG。点赞用 👍，在看固定用 👀，转发用简洁箭头符号（如 ➤）。
点赞和在看按钮：纯白背景+微弱灰色软阴影。
转发按钮：主题色 10% 透明度浅底 + 微弱主题色发光阴影，"转发"文字加粗。
底部致谢语固定使用：{followCardFooterText}（如不提供，则不生成底部致谢语区域）
底部致谢语居中显示。若是中文，用大字号浅灰居中留白收尾。
```
