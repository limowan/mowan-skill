# 动效设计思考框架

## 核心原则

每个动效必须回答三个问题：

1. **观众需要理解什么？**（信息目标）
2. **什么视觉方式最直观地传达这个信息？**（表达匹配）
3. **什么工具最快实现？**（实现效率）

从"观众需要看到什么"出发，不是从"我会什么效果"出发。

---

## 按信息类型思考视觉方向

下表是常见方向参考，**不是全部选项，不是必须使用的菜单**。如果内容需要一种没见过的表达方式，就自己设计。

| 要传达的信息 | 可以考虑的视觉方向（不限于此） |
|-------------|-------------------------------|
| 概念首次出现 | 揭示、聚光灯、zoom in、模糊变清晰、从暗到亮、文字生长 |
| 流程 / 步骤 | 路径追踪、时间线、进度条、逐步组装、管道流动、节点连线 |
| 对比 / 选择 | 分屏、天平、卡片翻转、高亮差异、滑块切换、前后对照 |
| 数据变化 | 图表动画、数字滚动、面积变形、颜色渐变、柱状生长、环形填充 |
| 因果关系 | 连线、多米诺、链式反应、箭头流、触发-响应、涟漪扩散 |
| 强调 / 金句 | 打字机、手写体、光扫、弹性放大、抖动、下划线绘制、标记笔高亮 |
| 结构 / 组成 | 零件组装、爆炸图、分层展开、嵌套、拼图、模块飞入 |
| 氛围 / 情绪 | 粒子系统、渐变背景、循环动画、光影变化、波纹、呼吸感缩放 |

## 工具匹配思路

确定视觉方向后，再选最高效的工具：

- **Remotion**：适合完整场景、React 组件化、品牌化视觉系统、批量生成
- **Manim**：适合数学公式、算法步骤、几何变换、技术原理推导
- **HyperFrames**：适合图文排版、字幕、标题卡、快速转场
- **HTML/GSAP/SVG**：适合独立特效片段（打字机、聚光灯、SVG 组装、Logo reveal），做完导出透明素材合成进主视频
- **Three.js/Canvas**：适合 3D 场景、程序化背景、粒子系统、沉浸感画面
- **浏览器录制**：适合网页产品真实操作演示、交互流程展示

工具可以混合使用。一个视频里 Remotion 做主片、GSAP 做标题特效、Manim 做核心动画，完全没问题。

## 鼓励创造

- 上面的表格只是起点，不是边界
- 如果内容需要一种全新的视觉表达，搜索 Dribbble、motion design 参考，或者直接设计
- 可以组合多种方向：比如"零件组装 + 聚光灯揭示"、"路径追踪 + 数字滚动"
- 如果已有专用动效 Skill（如 svg-assembly-animator、light-spotlight-render）能更快生成效果，优先调用；没有就按当前项目临时实现

## 反模式

- 信息已经清楚的画面不需要加动效，静态展示就够了
- 不要为了炫技加动效：每个动效必须服务理解、记忆点或节奏变化
- 不要全片高密度动效，要有呼吸感——有动有静才有节奏
- 不要所有段落用同一种动效，视觉疲劳比没有动效更糟
