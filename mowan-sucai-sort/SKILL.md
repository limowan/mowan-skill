# 墨玩素材整理助手 · 工作流编排

本文件是 Skill 的**流程编排器**。具体的分类规则存放在 `resources/` 文件夹中。

执行本 Skill 时，**必须先读取对应的资源文件**再开始整理。

---

## 资源文件索引

| 文件 | 内容 | 何时读取 |
|------|------|---------|
| `resources/分类规则.md` | 各类素材的分类维度和规则说明 | 识别素材类型后 |
| `resources/索引模板.md` | 各类素材的索引条目格式模板 | 创建索引时 |

---

## 执行流程

### 第一步：自动探测或询问素材库路径

**自动探测常见位置：**
1. 当前工作目录下的常见文件夹名：
   - `素材库/`、`我的素材/`、`素材/`、`notes/`、`library/`、`resources/`、`资料库/`
2. 用户主目录下的特定文件夹
3. 环境变量或配置文件中指定的路径

**探测结果处理：**
- 如果找到唯一匹配的文件夹 → 直接采用，向用户确认：「找到素材库位置：[路径]，是否使用？」
- 如果找到多个 → 列出供用户选择
- 如果未找到 → 询问用户：「请提供你的素材库根目录路径（或使用当前目录）」

**记录路径：**
将用户确认的路径写入内存，本次会话持续使用。

---

### 第二步：首次初始化流程（检测是否为新用户）

**检测素材库结构：**
扫描用户指定的素材库根目录，检查是否存在以下特征：
- 已有「内容文件夹 + 分类索引文件夹」的配对结构
- 存在 `_config.md` 配置文件
- 存在 `素材自动更新记录/` 目录

**如果检测为新用户（无上述特征）：**

1. **询问素材类型**（多选）：
   ```
   检测到这是首次使用。你需要整理哪些类型的素材？
   
   [ ] 微信读书笔记（从微信读书App导出的笔记和摘录）
   [ ] 公众号文章（自己写的或收藏的公众号文章）
   [ ] 小红书笔记（发布的或收藏的图文笔记）
   [ ] 个人复盘/日记（工作复盘、生活记录等）
   [ ] 网页摘录（从网页保存的文章、划线等）
   [ ] 其他（自定义类型）
   ```

2. **为每个选中的类型询问详情：**
   - 素材存放位置：
     - 选项1：「使用默认位置：[素材库根目录]/[类型名]/」
     - 选项2：「已存在，路径是：_____」
     - 选项3：「在根目录下创建新文件夹，命名为：_____」
   
   - 分类维度（提供默认值，用户可修改）：
     - 微信读书笔记：按书籍分类、按主题分类、按阅读时间分类
     - 公众号文章：按主题分类、按发布时间分类
     - 小红书笔记：按目的地/主题分类、按发布时间分类
     - 个人复盘：按时间分类（周/月/年）、按主题分类

3. **自动创建目录结构：**
   ```
   [素材库根目录]/
   ├── [类型1]/
   │   ├── [内容文件夹]/          ← 存放原始素材
   │   └── [分类索引文件夹]/       ← AI生成的索引（自动创建）
   ├── [类型2]/
   │   ├── ...
   │   └── ...
   └── 素材自动更新记录/           ← 更新日志
   ```

4. **创建 `_config.md` 配置文件：**
   每个分类索引文件夹下创建，记录对应的内容文件夹路径。

---

### 第三步：扫描并发现目录结构

**每次使用前必须执行此步骤。**

1. 递归扫描素材库根目录（两层深度），获取所有文件夹列表
2. 自动识别「内容文件夹 + 分类索引文件夹」配对关系
3. 检测是否有新增的内容文件夹（没有对应分类索引文件夹的）
4. 检测是否有路径变动（文件夹改名、移动等）

**识别规则：**
- 分类索引文件夹特征：名称中包含"分类"、"索引"或"_index"
- 内容文件夹特征：与某个分类文件夹名称有对应关系
- 特殊文件夹排除：`素材自动更新记录/`、`.obsidian/`、`.git/` 等非素材目录

**路径变动检测：**
- 读取上次更新日志，获取上次记录的目录结构
- 对比当前扫描结果，发现新增/重命名/移动的文件夹
- 如有变动，向用户确认

---

### 第四步：确认操作模式

根据扫描结果，自动判断或询问用户：

- **自动发现新文件夹** → 新增模式（为新文件夹创建分类索引）
- **已有文件夹内容变化** → 更新模式（增量更新索引）
- **用户明确指定** → 按用户要求操作

---

### 第五步：读取资源文件并识别素材类型

**读取 `resources/分类规则.md`：**
根据文件夹名称识别素材类型：

| 素材类型 | 识别特征 | 使用的规则 |
|---------|---------|-----------|
| 微信读书笔记 | 文件夹名含"微信读书"或"读书笔记" | 微信读书笔记分类规则 |
| 公众号文章 | 文件夹名含"公众号" | 公众号文章分类规则 |
| 小红书笔记 | 文件夹名含"小红书"、"旅行"或"攻略" | 旅游攻略分类规则 |
| 个人复盘 | 文件夹名含"复盘"、"日记"或"journal" | 个人复盘分类规则 |
| 网页摘录 | 文件夹名含"摘录"、"剪藏"或"webclip" | 网页摘录分类规则 |
| 其他类型 | 无法匹配 | 询问用户后自定义 |

**遇到新类型的处理：**
- 询问用户分类维度和规则
- 将新规则追加到 `resources/分类规则.md`

---

### 第六步：创建/更新分类索引

**新增模式：**
1. 创建分类索引文件夹（如果不存在）
2. 创建 `_config.md` 配置文件，写入根目录路径
3. 扫描内容文件夹中的所有文件
4. 逐个分析文件内容，生成分类索引（只写文件名，不写根目录路径）
5. 写入索引文件

**更新模式**（基于文件修改日期）：
1. 扫描内容文件夹，获取每个文件的**最后修改时间**
2. 读取上次更新日志的时间戳
3. 找出修改时间晚于上次更新时间的文件（即新增或修改的文件）
4. 对这些文件进行增量更新，在对应分类索引**末尾追加**新条目

**获取文件修改日期（跨平台）：**
- Windows (PowerShell): `Get-ChildItem -Path "[路径]" -Filter "*.md" | Select-Object Name, LastWriteTime`
- Unix/macOS (Bash): `ls -lt --time-style=full-iso "[路径]"/*.md`
- 通用：使用 Node.js 的 `fs.stat()` 或 Python 的 `os.path.getmtime()`

**索引条目格式（读取 `resources/索引模板.md`）：**
- 索引中的每条记录**只写文件名**，不写根目录路径
- 检索时通过 `_config.md 中的根目录 + / + 文件名` 拼接出完整路径

---

### 第七步：记录更新日志

**日志存储路径：**
```
[素材库根目录]/素材自动更新记录/[素材类型]素材更新记录/
```

**文件命名：** `YYYY-MM-DD_HH-MM_更新日志.md`

**时间获取：** 使用本地时间

**日志内容必须包含：**
1. 更新时间
2. 素材类型
3. 素材文件夹路径（当前实际路径）
4. 分类文件夹路径（当前实际路径）
5. 变更统计（新增/删除/修改数量）
6. 新增文件详情（文件名、归入分类、核心观点）
7. 目录结构快照（记录当前素材库完整目录结构，用于下次对比）

---

整理完成后，输出 Markdown 格式的总结报告：

## 索引格式规范

### `_config.md` 配置文件

每个分类索引文件夹的根目录下必须有一个 `_config.md`：

```markdown
# 索引配置

根目录: [相对于素材库根目录的内容文件夹路径]
```

### 索引条目格式

**微信读书笔记类：**
```markdown
### 总索引

- 《书名》阅读笔记.md
  - 一句话简介：简要概括（分类：主题分类）
- 《另一本书》摘录.md
  - 一句话简介：简要概括（分类：主题分类）
```

**公众号文章类：**
```markdown
### 主题分类

- [日期]文章标题.md
  - 核心观点：一句话概括
- [日期]另一篇文章.md
  - 核心观点：一句话概括
```

### 根目录变更处理

当内容文件夹改名时，**只需修改 `_config.md` 中的根目录**，所有索引条目无需改动。

## 用户确认规范

**所有需要向用户确认的环节，必须优先调用 `ask question` 类工具进行结构化提问**（选项由AI根据具体场景灵活设置，可包含用户自由填写选项）。如果当前环境无此类工具，则以文字方式向用户提出明确的问题并等待回复。

需要向用户确认的场景包括但不限于：
- 发现多个可能的素材库位置时选择
- 首次使用选择素材类型
- 发现目录结构变动（新增/重命名/移动），需要确认是否正确
- 新增素材类型，需要确认分类维度和规则
- 整理完成后的总结确认
- 任何存在歧义或不确定的操作

---

## 注意事项

1. **不修改原始素材**：只创建/更新分类索引文件夹，不动原始素材文件
2. **基于文件修改日期判断**：更新模式下，用 `LastWriteTime` 判断哪些文件需要处理
3. **自适应目录结构**：每次使用前扫描目录，不硬编码路径
4. **路径变动检测**：对比上次日志中的目录结构快照，发现变动后向用户确认
5. **索引使用 _config.md + 子路径格式**：索引条目只写文件名，根目录统一在 `_config.md` 中定义
6. **根目录变更时只改 _config.md**：内容文件夹改名时，只需更新 `_config.md`，不需要修改任何索引条目
7. **新增条目追加到分类文件末尾**：更新模式下无需重写整个索引文件
8. **询问新类型**：遇到未知素材类型时，主动询问用户定义规则
9. **日志中记录目录快照**：便于下次对比发现变动
10. **用户确认必须使用工具**：所有需要用户确认的环节，优先使用 ask question 工具，无法调用时使用文字提问
