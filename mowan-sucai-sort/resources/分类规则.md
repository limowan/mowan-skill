# 素材分类规则

本文件定义各类素材的分类维度和规则。

---

## 微信读书笔记

### 分类维度建议

1. **按书籍分类**（默认）
   - 每本书一个分类
   - 适合：阅读书籍较多，按书管理

2. **按主题分类**
   - 商业管理、个人成长、心理学、历史人文、科技趋势等
   - 适合：跨书籍的主题研究

3. **按阅读时间分类**
   - 2025年阅读、2024年阅读等
   - 适合：追踪阅读进度和年度回顾

### 索引条目格式

```markdown
- 《书名》阅读笔记.md
  - 一句话简介：简要概括核心内容（分类：所属分类）
  - 阅读时间：2025-01-15
  - 标签：#标签1 #标签2
```

### 示例

```markdown
### 商业管理

- 《从0到1》读书笔记.md
  - 一句话简介：彼得·蒂尔关于创业公司如何打造垄断优势的核心观点（分类：商业管理）
  - 阅读时间：2025-01-10
  - 标签：#创业 #商业战略

- 《纳瓦尔宝典》摘录.md
  - 一句话简介：关于财富积累和幸福人生的智慧合集（分类：商业管理）
  - 阅读时间：2025-02-20
  - 标签：#财富 #人生哲学

### 个人成长

- 《深度工作》阅读笔记.md
  - 一句话简介：在碎片化时代如何保持专注和高质量产出的方法（分类：个人成长）
  - 阅读时间：2025-03-05
  - 标签：#效率 #专注力
```

## 公众号文章

### 分类维度建议

1. **按主题分类**（推荐）
   - AI工具效率、个人成长、职场经验、商业观察、生活感悟等
   - 适合：内容主题多样的博主

2. **按发布时间分类**
   - 2025年文章、2024年文章、月度归档等
   - 适合：时间线明确的输出型博主

3. **按系列/专栏分类**
   - 专栏A、专栏B、系列文章等
   - 适合：有固定专栏的博主

### 索引条目格式

```markdown
- [YYYY-MM-DD]文章标题.md
  - 核心观点：一句话概括文章主旨
  - 标签：#标签1 #标签2
```

### 示例

```markdown
### AI工具效率

- [2025-03-15]用Claude自动整理素材库的方法.md
  - 核心观点：通过分类索引建立个人知识管理体系，实现素材的快速检索和复用
  - 标签：#Claude #素材管理 #AI工作流

- [2025-04-01]10个提升写作效率的Prompt技巧.md
  - 核心观点：结构化Prompt设计原则，让AI输出更符合预期的内容
  - 标签：#Prompt #写作 #效率

### 个人成长

- [2025-02-28]我如何用日复盘实现持续进步.md
  - 核心观点：日复盘不是记录流水账，而是建立反馈闭环的关键习惯
  - 标签：#复盘 #习惯 #成长
```

## 小红书笔记/旅游攻略

### 分类维度建议

1. **按目的地分类**（旅游类推荐）
   - 国内：城市名或省份
   - 国外：国家或地区

2. **按主题分类**
   - 美食探店、旅行攻略、生活记录、好物分享等

3. **按发布时间分类**
   - 2025年、2024年等

### 索引条目格式

```markdown
- 笔记标题.md
  - 目的地/主题：简要说明
  - 类型：自由行/跟团/美食/探店等
  - 关键词：关键词1, 关键词2, 关键词3
```

### 示例

```markdown
### 国内旅行

- 国庆天津旅游（保姆级）攻略.md
  - 目的地：天津
  - 类型：自由行/城市综合
  - 天数：2天
  - 关键词：五大道, 海河夜景, 民国故居

- 成都美食地图：本地人私藏小店.md
  - 目的地：成都
  - 类型：美食探店
  - 关键词：火锅, 串串, 川菜, 小吃

### 出境游

- 日本关西7日深度游攻略.md
  - 目的地：日本关西（京都/大阪/奈良）
  - 类型：自由行
  - 天数：7天
  - 关键词：樱花季, 寺庙, 怀石料理
```

## 个人复盘/日记

### 分类维度建议

1. **按时间分类**（推荐）
   - 周复盘、月复盘、年复盘
   - 按年度分文件夹：2025年、2024年

2. **按主题分类**
   - 工作复盘、学习复盘、生活复盘、项目复盘等

3. **按项目分类**
   - 项目A复盘、项目B复盘等

### 索引条目格式

```markdown
- 复盘标题.md
  - 时间范围：YYYY-MM-DD ~ YYYY-MM-DD
  - 复盘类型：周/月/年/项目
  - 核心收获：一句话概括
```

### 示例

```markdown
### 2025年复盘

- 2025-W12 周复盘.md
  - 时间范围：2025-03-17 ~ 2025-03-23
  - 复盘类型：周复盘
  - 核心收获：建立了素材整理自动化流程，效率提升显著

- 2025-03 月复盘.md
  - 时间范围：2025-03-01 ~ 2025-03-31
  - 复盘类型：月复盘
  - 核心收获：完成3篇文章输出，启动个人知识库建设项目

### 项目复盘

- 小红书账号从0到1复盘.md
  - 时间范围：2025-01-01 ~ 2025-03-31
  - 复盘类型：项目复盘
  - 核心收获：内容垂直度是关键，选题要兼顾热点与差异化
```

## 网页摘录/剪藏

### 分类维度建议

1. **按主题分类**（推荐）
   - 与公众号文章类似：AI工具、产品思维、设计灵感、行业报告等

2. **按来源分类**
   - 公众号、知乎、少数派、Product Hunt等

3. **按阅读状态分类**
   - 待阅读、已阅读、重点收藏

### 索引条目格式

```markdown
- 文章标题.md
  - 原文链接：[URL]
  - 来源：网站/作者
  - 一句话简介：核心观点概括
  - 标签：#标签1 #标签2
```

### 示例

```markdown
### AI与产品

- 2024年AI产品设计的10个趋势.md
  - 原文链接：[URL]
  - 来源：少数派/作者名
  - 一句话简介：AI产品从功能导向转向意图导向，自然语言成为主流交互方式
  - 标签：#AI产品 #设计趋势

- ChatGPT Prompt Engineering最佳实践.md
  - 原文链接：[URL]
  - 来源：OpenAI官方文档
  - 一句话简介：系统提示词设计、少样本示例、链式思考等核心技巧
  - 标签：#Prompt #ChatGPT #官方文档
```

## 自定义类型

当遇到新的素材类型时，询问用户以下问题来建立分类规则：

1. **这是什么类型的素材？**（给出一个清晰的类型名称）
2. **你希望如何分类？**（提供2-3个建议，让用户选择或自定义）
3. **索引条目需要包含哪些信息？**（如：标题、日期、标签、核心观点等）

然后将新类型的分类规则追加到本文件中，格式如下：

```markdown
## [新类型名称]

### 分类维度建议

1. [维度1]
2. [维度2]
...

### 索引条目格式

```markdown
- 文件名.md
  - 字段1：值
  - 字段2：值
```

### 示例

[具体示例]
```
