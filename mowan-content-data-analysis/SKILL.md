# 内容数据分析助手

## 核心功能

拆解别人的爆款内容（公众号文章、小红书笔记等），提炼可学习的写作技巧，并与用户自己的同主题内容做横向对比，输出具体的差距分析和改进建议。

## 适用场景

- 看到一篇爆款文章/笔记，想知道它为什么火
- 想学习某个博主的内容风格和技巧
- 写完一篇文章后，想跟同主题的爆款对比找差距
- 定期做内容复盘，分析自己和头部内容的差异

## 工作流程

### 步骤1：获取分析对象

用户可能提供以下形式的输入：
- 公众号文章链接 → 调用 `wechat-article-downloader` 或 `defuddle` 提取内容
- 小红书笔记链接 → 调用 `defuddle` 提取内容
- 直接粘贴的文章/笔记内容
- 截图（小红书笔记截图等）

**确认信息**：
```
请确认以下信息：
1. 分析对象：[文章标题/链接]
2. 平台类型：[公众号/小红书/其他]
3. 是否需要与你自己的内容做对比？
```

### 步骤2：内容拆解分析

根据平台类型，从以下维度拆解内容：

#### 通用维度（所有平台）

| 维度 | 分析内容 |
|------|---------|
| **选题角度** | 切入点是什么？为什么这个角度能吸引人？是信息差、情绪共鸣、还是实用价值？ |
| **标题/封面** | 标题用了什么技巧？（悬念、数字、痛点、反常识等）封面设计有什么特点？ |
| **开头钩子** | 前3句话用了什么钩子？能不能在3秒内抓住注意力？ |
| **内容结构** | 整体结构是什么？（总分总/递进/对比/故事线）每个板块的作用是什么？ |
| **核心价值** | 读者看完能得到什么？（知识/方法/情感共鸣/谈资）HKR评估 |
| **语言风格** | 口语化程度、金句密度、人味程度、情绪表达方式 |
| **互动设计** | 有没有引导互动的设计？（提问、投票、征集、CTA） |

#### 公众号专项

| 维度 | 分析内容 |
|------|---------|
| **排版节奏** | 段落长度、留白、加粗使用、图文比例 |
| **金句密度** | 每千字有几个值得摘录的金句？金句的类型（观点型/比喻型/反转型） |
| **叙事技巧** | 有没有用故事包裹观点？故事的完整度和代入感如何？ |
| **认知翻转** | 有没有挑战读者的既有认知？翻转的力度和说服力如何？ |
| **文化升维** | 有没有从具体事件连接到更大的文化/哲学/历史参照？ |

#### 小红书专项

| 维度 | 分析内容 |
|------|---------|
| **封面图** | 构图、配色、文字排版、信息层级 |
| **标题公式** | 用了哪种标题公式？（数字+痛点、反问、对比、身份标签等） |
| **内容密度** | 信息量与篇幅的比例，是否做到了"短而有料" |
| **标签策略** | 话题标签的选择逻辑，覆盖了哪些流量池 |
| **评论区运营** | 置顶评论、互动引导、评论区内容补充 |

### 步骤3：提炼可学习的技巧

从拆解结果中提炼出具体的、可复用的技巧清单：

```markdown
## 可学习的技巧

### 立刻能用的（下一篇就能试）
1. **[技巧名]**：[具体描述 + 原文示例]
   - 怎么用：[一句话说清楚操作方法]

### 需要练习的（长期积累）
1. **[技巧名]**：[具体描述]
   - 练习方法：[怎么刻意练习这个技巧]
```

### 步骤4：横向对比（默认执行）

每次分析都必须与用户自己的同主题内容做横向对比，这是本 Skill 的核心价值——不只是拆解别人的内容，更要找到自己和爆款之间的具体差距。执行以下流程：

1. **搜索用户素材**：调用 `sucai-sousuo`，从用户的所有素材类型中搜索同主题或相近主题的内容
2. **选择对比对象**：找到最接近的 1-2 篇用户自己的文章/笔记
3. **逐维度对比**：

```markdown
## 横向对比

**对比对象**：
- 爆款：《[标题]》
- 你的：《[标题]》

| 维度 | 爆款做法 | 你的做法 | 差距分析 | 改进建议 |
|------|---------|---------|---------|---------|
| 选题角度 | ... | ... | ... | ... |
| 标题 | ... | ... | ... | ... |
| 开头钩子 | ... | ... | ... | ... |
| 内容结构 | ... | ... | ... | ... |
| 核心价值 | ... | ... | ... | ... |
| 语言风格 | ... | ... | ... | ... |
| 金句密度 | ... | ... | ... | ... |
```

4. **差距总结**：

```markdown
## 差距总结

### 最大的差距（优先改进）
1. **[差距点]**：[具体描述，不要笼统]
   - 爆款怎么做的：[具体引用]
   - 你可以怎么改：[具体建议]

### 你做得好的地方（继续保持）
1. **[优势点]**：[具体描述]

### 一句话总结
[用一句话概括核心差距和改进方向]
```

### 步骤5：输出分析报告

```markdown
# 内容分析报告

**分析对象**：[标题]
**平台**：[公众号/小红书]
**分析日期**：[日期]

---

## 1. 内容拆解
[步骤2的输出]

## 2. 可学习的技巧
[步骤3的输出]

## 3. 横向对比
[步骤4的输出]

---

## 下一步行动
- [ ] [具体的、可执行的改进动作1]
- [ ] [具体的、可执行的改进动作2]
- [ ] [具体的、可执行的改进动作3]
```

## 与其他 Skill 的协作

```
mowan-content-data-analysis（主调度）
    ├── wechat-article-downloader → 提取公众号文章内容
    ├── defuddle → 提取网页/小红书内容
    ├── sucai-sousuo → 搜索用户自己的同主题素材做对比
    ├── xiaohongshu-note-analyzer → 小红书笔记深度分析（可选，补充分析维度）
    └── x-blogger-analyzer → X/Twitter 博主分析（可选）
```

## 注意事项

1. **客观分析，不吹不黑**：爆款不一定每个维度都好，该指出问题就指出
2. **对比要公平**：考虑发布时间、粉丝基数、平台算法等因素的差异
3. **建议要具体**：不要说"标题要更吸引人"，要说"可以试试用数字+痛点的公式，比如把'xxx'改成'xxx'"
4. **尊重用户风格**：对比的目的是学习技巧，不是让用户变成别人。建议要在保持用户自身风格的基础上改进
5. **不要过度解读**：有些内容火是因为时机、运气、粉丝基数，不全是技巧的功劳。要诚实地指出这一点
